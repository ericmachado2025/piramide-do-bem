## 5. SÉRIES FILTRADAS POR SCHOOL_TYPE

A lista de séries mostra TODAS (Pré I até 3º EM) independente da escola. 

**Correção:** Após selecionar a escola, verificar o `school_type` da escola e filtrar:
- `infantil` ou `creche` → Pré I, Pré II
- `fundamental` → 1º ao 9º ano (se o nome da escola contém "EMEF" ou escola é de fundamental)
- `medio` → 1º ao 3º EM
- `tecnico` → 1º ao 4º Técnico
- `superior` → 1º ao 10º semestre
- Se school_type contém múltiplos (ex: "fundamental,medio") → mostrar ambos

**PROBLEMA ADICIONAL:** Todas as 180.230 escolas estão com school_type='fundamental'. Isso precisa ser corrigido no banco. Fazer UPDATE baseado no nome da escola:
```sql
-- Escolas que têm "EM " ou "EEEM" ou "MEDIO" no nome → medio
UPDATE schools SET school_type = 'medio' 
WHERE (name ILIKE '%ENSINO MEDIO%' OR name ILIKE '%EEEM%' OR name ILIKE '% EM %') 
AND school_type = 'fundamental';

-- Escolas que têm "EEI" ou "EMEI" ou "CRECHE" ou "INFANTIL" → infantil
UPDATE schools SET school_type = 'infantil'
WHERE (name ILIKE '%INFANTIL%' OR name ILIKE '%EMEI%' OR name ILIKE '%EEI %' OR name ILIKE '%CRECHE%')
AND school_type = 'fundamental';

-- Universidades e faculdades → superior
UPDATE schools SET school_type = 'superior'
WHERE (name ILIKE '%UNIVERSIDADE%' OR name ILIKE '%FACULDADE%' OR name ILIKE '%INSTITUTO FEDERAL%')
AND school_type = 'fundamental';

-- EJA
UPDATE schools SET school_type = 'eja'
WHERE name ILIKE '%EJA%'
AND school_type = 'fundamental';

-- Técnico
UPDATE schools SET school_type = 'tecnico'
WHERE (name ILIKE '%TECNIC%' OR name ILIKE '%CEFET%' OR name ILIKE '%ETEC%')
AND school_type = 'fundamental';
```

## 10. BACK DO NAVEGADOR

O botão de voltar do navegador (browser back) retorna para o início em vez de voltar 1 passo.

**Correção:** Usar `window.history.pushState` a cada mudança de step no cadastro e na escolha de tribo. Adicionar listener para `popstate`:

```typescript
useEffect(() => {
  const handlePopState = (e: PopStateEvent) => {
    e.preventDefault()
    if (stepIndex > 0) {
      setStepIndex(stepIndex - 1)
    } else {
      navigate('/')
    }
  }
  window.addEventListener('popstate', handlePopState)
  return () => window.removeEventListener('popstate', handlePopState)
}, [stepIndex])

// A cada mudança de step:
useEffect(() => {
  window.history.pushState({ step: stepIndex }, '')
}, [stepIndex])
```

## 11. DISPLAY_ORDER DINÂMICO

Criar mecanismo que periodicamente reordena as listas baseado na popularidade:

Para categories, types, communities:
```sql
-- Função que reordena baseado na contagem de alunos
CREATE OR REPLACE FUNCTION update_display_orders() RETURNS void AS $$
BEGIN
  -- Reordenar categories pelo número de alunos
  WITH ranked AS (
    SELECT cc.id, ROW_NUMBER() OVER (ORDER BY COUNT(s.id) DESC) as new_order
    FROM community_categories cc
    LEFT JOIN community_types ct ON ct.category_id = cc.id
    LEFT JOIN communities c ON c.type_id = ct.id
    LEFT JOIN students s ON s.community_id = c.id
    GROUP BY cc.id
  )
  UPDATE community_categories SET display_order = r.new_order
  FROM ranked r WHERE community_categories.id = r.id;
  
  -- Similar para types e communities
END;
$$ LANGUAGE plpgsql;
```
