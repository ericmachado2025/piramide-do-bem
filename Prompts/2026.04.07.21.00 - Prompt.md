2. Converter para INSERTs PostgreSQL na tabela `community_levels`:
   - O `id` deve ser o PublicId (UUID) do SQL Server
   - O `community_id` deve ser o UUID da community correspondente (já existe no Supabase — fazer JOIN pelo PublicId da community no Communities.sql original)

### 1.3 Vincular characters aos levels corretos

Cada character no Communities.sql tem um LevelId (SnowflakeId) que referencia um Level. Precisa:

1. Criar mapeamento: SnowflakeId do Level → UUID (PublicId) do Level
2. Criar mapeamento: SnowflakeId do Character → UUID (PublicId) do Character  
3. Para cada character, encontrar o level correto pelo mapeamento de SnowflakeIds
4. UPDATE characters SET level_id = [UUID do level correto]

**Estratégia mais simples (se o mapeamento direto for complexo):**
Usar a relação community_id + tier. Cada character pertence a uma community e tem um display_order que indica seu tier:
- display_order 1-5 → Tier 1 (level com tier=1 da mesma community)
- display_order 6-10 → Tier 2
- display_order 11-15 → Tier 3
- display_order 16-20 → Tier 4
- display_order 21-25 → Tier 5

Mas como cada community pode ter números diferentes de characters por tier (3-8), a estratégia mais precisa é:

1. Do Communities_levels.sql, extrair pares: (Level_SnowflakeId, Level_PublicId)
2. Do Communities.sql, para cada character, extrair o LevelId (SnowflakeId) — é o 4º campo nos VALUES
3. Mapear: character.level_id = level.public_id WHERE level.snowflake_id = character.original_level_snowflake_id

Gerar o SQL de UPDATE e executar via supabase db query.

## BLOCO 3 — Refatorar código para usar scoring_rules

### 3.1 Criar função no database.ts:
```typescript
// Buscar pontos de uma ação pela rule_key
export async function getScoringRule(ruleKey: string) {
  const { data } = await supabase
    .from('scoring_rules')
    .select('*')
    .eq('rule_key', ruleKey)
    .eq('active', true)
    .single()
  return data
}

// Buscar todas as regras ativas de ações
export async function getActionScoringRules() {
  const { data } = await supabase
    .from('scoring_rules')
    .select('*')
    .eq('rule_type', 'action')
    .eq('active', true)
    .order('points', { ascending: true })
  return data ?? []
}
```
